# CC v0 Scope

Date: 2026-04-09
Status: proposed first concrete product direction

## Product direction

CC v0 is a lightweight chess challenge tracker for Lichess players who want a fast, slightly absurd skill loop: pick one concrete challenge, play a real game, and come back to see whether the run counted.

## Target user

A Lichess player who is willing to sign in, save a username, and complete one simple challenge without needing coaching, social features, or deep analytics.

## First user promise

Save your Lichess username, start one clear challenge, submit a finished game, and get a simple pass/fail result tied to your real game.

## Minimum shippable v0 loop

1. Sign in with Clerk.
2. Save a Lichess username in account settings.
3. View a challenge list with the starter challenges already defined in the repo.
4. Open a single challenge detail page.
5. Start that challenge and persist active challenge state on the signed-in user.
6. Submit a finished Lichess game ID for that challenge.
7. Show a basic result state for the submitted attempt: pending, passed, or failed/manual review placeholder.

## In scope for v0

- `/account` route that lets the user save or update their Lichess username.
- `/challenges` route that lists the starter challenges.
- `/challenges/[id]` route that explains one challenge and lets the user start or submit it.
- Clerk-backed per-user metadata for username, active challenge, and simple progress state.
- One end-to-end challenge proof loop that is reviewable inside the app.

## Explicitly out of scope

- Automated Lichess API verification beyond a minimal placeholder/manual-review result.
- Challenge creation by users or admins.
- Leaderboards, friends, chat, feeds, or multiplayer competition mechanics.
- Payments, rewards redemption, or marketplace features.
- Mobile app packaging.
- Rich analytics, coaching, engine analysis, or opening prep features.
- Multi-provider auth beyond Clerk.

## Why this scope first

The current homepage already promises this exact loop, but the linked routes are missing. Shipping the smallest real route set and one proof-bearing challenge flow is the fastest way to turn the existing front door into a usable product.
